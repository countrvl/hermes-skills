# Prompt Optimizer

## Overview

Двухрежимный инструмент для работы с промптами:

- **Режим A — «Создать»**: по идее пользователя задаёт уточняющие вопросы и генерирует структурированный промпт.
- **Режим Б — «Оптимизировать»**: прогоняет готовый промпт через три параллельных чекера (противоречия, формат, few-shot), показывает проблемы и после подтверждения исправляет.

Основан на методологии OpenAI Cookbook «Optimize Prompts» — multi-agent pipeline с разделением на detection и rewriting фазы.

Рантайм: `scripts/optimize.py` — Python-скрипт, который сам ходит в LLM API (OpenAI-совместимое) и выполняет проверки параллельно.

## When to Use

- Пользователь говорит: «помоги написать промпт для X», «создай промпт», «придумай промпт»
- Пользователь говорит: «оптимизируй промпт», «проверь промпт», «улучши промпт», «промпт криво работает»
- Пользователь даёт готовый промпт и просит найти в нём проблемы
- Пользователь хочет проверить few-shot примеры на соответствие промпту

**Не используй для:**
- Простого перевода промпта на другой язык (это не оптимизация)
- Написания кода, документации, статей (это не промпты)
- Отладки конкретного бага в ответах модели (нужен systematic-debugging)

## Prerequisites

Скрипт `scripts/optimize.py` требует:

1. **Python 3.8+** с `httpx` (установи: `pip install httpx`)
2. **API-ключ** к OpenAI-совместимому провайдеру в переменной окружения:
   - `PROMPT_OPTIMIZER_API_KEY` (приоритет)
   - или `OPENAI_API_KEY`
   - или `HERMES_API_KEY`
3. Опционально:
   - `PROMPT_OPTIMIZER_API_BASE` — base URL (по умолчанию `https://api.deepseek.com/v1`)
   - `PROMPT_OPTIMIZER_MODEL` — модель (по умолчанию `deepseek-chat`)

Проверь доступность перед первым использованием:
```
python scripts/optimize.py --help
```

## Workflow A: Создание промпта с нуля

1. **Получи идею** от пользователя. Примеры: «Нужен промпт для код-ревью», «Хочу промпт для суммаризации документов», «Промпт для классификации обращений в техподдержку».

2. **Задай уточняющие вопросы** (через `clarify`, все сразу одним вызовом):
   ```
   1. Какую роль должна исполнять модель? (эксперт, ассистент, критик, ...)
   2. Что именно должно быть на выходе? Опиши структуру: формат, поля, ограничения.
   3. Есть ли строгие правила или запреты? Что нельзя делать ни в коем случае?
   4. Какой тон и стиль? (формальный/неформальный, кратко/подробно, ...)
   5. Есть ли примеры желаемого поведения? Если да — приложи.
   6. Контекст: кто конечный пользователь и в каком продукте/сценарии используется промпт?
   ```
   Не задавай вопросы по одному — все сразу. Дай варианты ответов для вопросов 1, 4, 5, 6.

3. **Сгенерируй промпт** на основе ответов. Структура:
   ```
   ## Role
   [кто ты, какую роль исполняешь]

   ## Task
   [что именно нужно сделать]

   ## Rules
   - [правило 1]
   - [правило 2]
   - [запреты]

   ## Output Format
   [структура вывода: JSON-схема, Markdown-шаблон, или свободный текст]

   ## Examples (опционально)
   [примеры запросов и ответов]
   ```

4. **Покажи результат** пользователю. Предложи перейти к оптимизации: «Проверить промпт на противоречия и проблемы формата?»

5. Если пользователь согласен — переходи в Workflow B, шаг 2 (проверка).

## Workflow B: Оптимизация готового промпта

### Шаг 1: Получи промпт

Пользователь даёт промпт (и опционально few-shot примеры). Если пользователь просто говорит «оптимизируй» без промпта — спроси.

### Шаг 2: Запусти проверку

Выполни:
```
python scripts/optimize.py check --prompt "..." [--examples '[...]']
```

Где:
- `--prompt` — полный текст промпта (экранируй кавычки)
- `--examples` (опционально) — JSON-массив сообщений `[{"role":"user","content":"..."},{"role":"assistant","content":"..."},...]`

Скрипт запустит три чекера параллельно и вернёт JSON:
```json
{
  "contradiction": {
    "has_issues": true,
    "issues": ["Инструкция требует одновременно X и Y"]
  },
  "format": {
    "has_issues": false,
    "issues": []
  },
  "fewshot": {
    "has_issues": true,
    "issues": ["Пример №1 не соответствует правилу Z"]
  },
  "has_any_issues": true
}
```

### Шаг 3: Покажи сводку пользователю

Отформатируй результат понятно:

```
## Результаты проверки

### Противоречия — найдено 2
1. Инструкция требует «отвечай только на английском», но также говорит «nunca respondas en inglés» (взаимоисключающие правила)
2. ...

### Формат вывода — проблем нет

### Few-shot примеры — найдено 1
1. Пример №1: ассистент дал развёрнутый ответ, хотя промпт требует «только да/нет»

---
**Исправить найденные проблемы?**
```

Затем спроси через `clarify`: «Исправить найденные проблемы?» с вариантами «Да, исправляй», «Только противоречия», «Нет, оставить как есть».

### Шаг 4: Исправление

Если пользователь подтвердил — передай результат проверки в скрипт рерайта:

```
python scripts/optimize.py rewrite --prompt "..." --check-result '<JSON_из_шага_2>' [--examples '[...]']

# Если чекер пропустил проблему, можно передать ручные проблемы:
python scripts/optimize.py rewrite --prompt "..." --issues '["Добавь явную JSON-схему ответа"]'

# Для точной маршрутизации используй секции contradiction/format/fewshot:
python scripts/optimize.py rewrite --prompt "..." --issues '{"contradiction":["Инструкции X и Y конфликтуют"],"format":["Не перечислены поля JSON"],"fewshot":["Пример №1 нарушает правило Z"]}' --examples '[...]'
```

Скрипт вернёт:
```json
{
  "new_prompt": "...",
  "new_examples": [...],
  "diff": "--- original\n+++ optimized\n@@ ... @@\n..."
}
```

### Шаг 5: Покажи результат

Выведи:
1. Unified diff (что изменилось)
2. Полный текст исправленного промпта
3. Если примеры тоже менялись — покажи изменения в примерах

## Checker and Rewriter Prompts (internal)

Эти промпты вшиты в `scripts/optimize.py`. При необходимости адаптации — правь скрипт.

### Contradiction Checker

Ищет логические противоречия: две инструкции, которые невозможно выполнить одновременно. Не флагает простые пересечения или избыточность.

### Format Checker

Определяет, требуется ли структурированный вывод (JSON/CSV/Markdown-таблица). Если да — проверяет, указаны ли поля, типы, структура. Действует консервативно.

### Few-shot Consistency Checker

Сверяет ответы ассистента в примерах с правилами из промпта. Флагает примеры, где ассистент нарушает инструкции.

### Rewriter

Хирургически исправляет только найденные проблемы. Сохраняет оригинальный стиль, тон и структуру. Не переписывает то, что работает.

## Common Pitfalls

1. **Кавычки в промпте ломают shell-команду.** Используй `execute_code` для вызова скрипта с длинными/сложными промптами, а не `terminal`.

2. **API-ключ не найден.** Проверь, что одна из переменных (`PROMPT_OPTIMIZER_API_KEY`, `OPENAI_API_KEY`, `HERMES_API_KEY`) установлена в окружении.

3. **httpx не установлен.** Выполни `pip install httpx`.

4. **Чекер не нашёл очевидной проблемы.** Чекеры консервативны по дизайну. Если пользователь уверен, что проблема есть — предложи ему описать её вручную и запустить рерайт с `--issues`. Аргумент принимает JSON-список строк или объект с секциями `contradiction`, `format`, `fewshot`; скрипт сам преобразует их в структуру `check_result`. Например: `--issues '["Добавь явную JSON-схему ответа"]'` или `--issues '{"format":["Не перечислены поля JSON"]}'`.

5. **Промпт слишком длинный.** Если промпт > 8000 символов — чекеры могут обрезать контекст. Предупреди пользователя и предложи разбить на части.

6. **Режим «Создать» без уточнений.** Не генерируй промпт без уточняющих вопросов — результат будет слишком общим.
